# 功能边界

SceneFab 是面向影视/短剧第一人称解说的生产工具，不是通用轨道剪辑软件。它把素材理解、脚本生成、配音字幕和平台导出串成一条标准化流程。

## 核心能力

| 能力 | 解决的问题 | 主要产物 |
| --- | --- | --- |
| 素材与集数管理 | 短剧批量素材容易混乱、来源不可追溯 | 素材清单、集数索引、输出目录 |
| 场景理解 | 人工逐段看素材效率低，容易漏掉冲突点 | 场景摘要、人物线索、桥段候选 |
| 短剧标签建模 | 题材、爽点、人物关系没有结构化输入 | 内容标签、关系备注、前情和下集钩子 |
| 第一人称脚本 | 解说稿容易变成第三人称复述 | Hook、主体、反击、收束和钩子 |
| 配音字幕 | 文案、语速、字幕安全区分离导致返工 | 配音音频、字幕、时间轴 |
| 平台导出 | 各平台画布、码率、字幕区域重复配置 | MP4 成片、剪映草稿、发布参数 |

## 适用场景

| 场景 | 推荐用法 |
| --- | --- |
| 短剧单集解说 | 录入题材、爽点、人物关系和集数上下文，优先生成 30-90 秒竖屏稿 |
| 短剧整季批量 | 先统一标签和关系设定，再按集生成脚本、配音和导出记录 |
| 电影/剧集片段解说 | 先做场景拆分和情绪峰值筛选，再生成第一人称视角脚本 |
| 剪映继续精剪 | 导出剪映草稿，保留字幕、配音和时间轴供人工二次处理 |

## 不适用场景

| 场景 | 原因 |
| --- | --- |
| 逐帧精修调色 | SceneFab 只负责生产链路，不替代专业调色软件 |
| 复杂多轨包装 | 多机位、复杂特效和商业包装建议进入专业剪辑工具继续处理 |
| 无版权素材搬运 | 项目要求素材来源可追溯，最终内容合规由使用者负责 |

## 生产输入

| 输入 | 为什么重要 |
| --- | --- |
| 素材来源 | 便于授权、复盘和异常定位 |
| 目标平台 | 影响字数、画布、字幕安全区和导出参数 |
| 题材标签 | 决定标题方向和叙事语气 |
| 爽点标签 | 决定 Hook 和高潮排序 |
| 人物关系 | 防止脚本前后矛盾 |
| 上一集摘要 | 避免重复解释已经发生的剧情 |
| 下一集钩子 | 提升连载内容的点击动机 |

## 质量门禁

| 维度 | 合格标准 |
| --- | --- |
| Hook | 前 3 秒出现冲突、身份、危险、目标或反转 |
| 桥段 | 短剧模式至少命中一个桥段标签 |
| 一致性 | 人物关系、前情和当前集剧情不冲突 |
| 字数 | 按平台时长控制在目标范围内 |
| 字幕 | 单条不超过两行，避开互动栏和标题区 |
| 音频 | 配音响度稳定，字幕与配音偏差目标小于 50ms |
