# 第一人称解说生产规范

本文档把公开平台规则、短视频制作规律和本项目工程约束合并为一套可执行流程。目标是让素材采集、脚本、配音、字幕、导出和发布都有统一标准。

## 核心原则

1. 短视频优先竖屏交付，默认画布为 1080×1920。
2. YouTube Shorts 已支持最长 3 分钟，脚本系统不能只按 60 秒模型估算字数。
3. 第一人称解说要用"我"的视角推动情绪和动机，不能只复述画面。
4. 短剧解说必须把题材标签、人物关系、冲突桥段和爽点作为结构化输入。
5. 字幕是移动端核心信息载体，必须和配音节奏一起设计。

## 短剧解说内容模型

短剧与电影解说最大的差异不是时长，而是"连续钩子密度"。热门内容通常直接暴露题材和情绪标签，生产系统也应把这些标签结构化。

| 结构字段 | 示例 | 用途 |
| --- | --- | --- |
| 题材 | 都市爱情 / 女性成长 / 家庭伦理 / 民国爱情 / 古风权谋 | 决定叙述语气和平台标签 |
| 爽点 | 打脸虐渣 / 逆袭 / 马甲 / 战神归来 / 追妻火葬场 | 决定 Hook 和高潮排序 |
| 人物关系 | 真千金-假千金 / 总裁-灰姑娘 / 婆媳 / 继母继姐 | 防止脚本前后矛盾 |
| 关键秘密 | 身世、婚约、怀孕、隐藏身份、系统能力 | 生成悬念和结尾钩子 |
| 集数上下文 | 第 N 集、上一集结尾、下一集预告 | 避免重复解释和断裂叙事 |

### 桥段识别优先级

1. **身份揭露**：谁其实是谁，谁隐瞒了什么。
2. **对峙冲突**：主角与反派/家人/恋人正面交锋。
3. **打脸反转**：前文羞辱在当前片段得到回击。
4. **救场**：关键人物、证据或能力突然出现。
5. **背叛**：信任关系破裂，推动情绪下坠。
6. **心动**：关系升温，适合甜宠/爱情线。
7. **结尾悬念**：下一集必须点击的未解决问题。

## 标准工作流

```text
素材导入
  → 版权/来源标注
  → 题材/爽点/人物关系标注
  → 场景拆分
  → 人物与关系抽取
  → 短剧桥段识别
  → 第一人称脚本生成
  → 配音与字幕对齐
  → 竖屏重构与安全区检查
  → 平台导出
  → 发布复盘
```

## 脚本标准

| 项目 | 标准 |
| --- | --- |
| 视角 | 默认第一人称，"我"是画面内角色或明确的代入叙述者 |
| 开头 | 前 3 秒必须给出冲突、身份、危险、目标或反转 |
| 信息密度 | 每 6-10 秒推进一次新信息，避免解释同一件事 |
| 句式 | 短句优先，保留口语停顿，避免书面总结腔 |
| 情绪 | 每段标注情绪强度，配音和字幕共用同一情绪标签 |
| 结尾 | 给出反转、悬念、后果或下一集钩子 |

### 短剧脚本结构

```text
Hook: 我以为自己只是被赶出家门，直到那份亲子鉴定摔在所有人面前。
关系: 我 / 养父母 / 假千金 / 真正支持我的人
冲突: 被羞辱、被误认、被迫离开
反击: 证据出现、身份揭露、反派失控
收束: 这一刻他们才知道，真正该走的人不是我。
下一集钩子: 可那个一直沉默的男人，竟然叫出了我的真名。
```

### 推荐时长与字数

| 成片时长 | 适用场景 | 中文解说稿参考 |
| --- | --- | --- |
| 30-45 秒 | 单冲突、单反转 | 120-180 字 |
| 60-90 秒 | 短剧单桥段 | 240-360 字 |
| 120-180 秒 | 多桥段串联 / Shorts 长版 | 480-720 字 |

## 视频交付标准

| 平台类型 | 画布 | 字幕安全区 | 推荐码率 |
| --- | --- | --- | --- |
| 抖音 / TikTok / Shorts / Reels | 1080×1920 | 上下各保留 12%，右侧保留互动栏 | 8-12 Mbps |
| B 站竖屏 / 小红书视频 | 1080×1920 | 底部保留标题与进度区 | 8-12 Mbps |
| 横屏补档 | 1920×1080 | 下方字幕 8-12% 区域 | 12-20 Mbps |

### 导出门禁

- 字幕不能为空，且单条字幕不超过两行。
- 字幕与配音偏差目标小于 50ms。
- 首屏 3 秒内必须出现人物、冲突或结果预告。
- 短剧成片必须至少命中一个桥段标签，并在结尾保留悬念或后果。
- 成片 loudness 不应忽高忽低，配音必须压过原声。
