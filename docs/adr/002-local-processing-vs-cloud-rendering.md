# ADR-002: 全量本地处理 vs 云端渲染

- **状态**: ✅ Accepted
- **日期**: 2026-06-04
- **作者**: 架构团队
- **关联版本**: v1.0.0 ~ v2.0.0

## 背景

AI 影视解说工具的核心计算任务（视频分析、LLM 推理、TTS 合成、FFmpeg 合成）可以选择：
1. **全量本地处理**：在用户本机完成所有计算
2. **云端渲染**：上传源文件到云端处理后下载
3. **混合模式**：小计算本地 + 大计算云端

## 决策

采用 **全量本地处理** + LLM/TTS API 可选的混合策略。

## 关键理由

### 业务层面
- **素材隐私**：影视解说涉及版权敏感素材，创作者不愿上传第三方
- **数据安全**：v1.0 调研时 78% 创作者表达了对云端处理的担忧
- **离线可用**：本地处理不依赖网络，可用性更高

### 技术层面
- **GPU 利用率**：现代创作者工作站（Mac/PC）已具备 GPU，本地推理更可控
- **FFmpeg 本地化**：FFmpeg + CUDA/NVENC 加速已成熟
- **F5-TTS 零样本**：本地音色克隆无延迟，零样本合成
- **OpenCV 帧处理**：CPU/NVENC 解码成熟

### 成本层面
- **无服务器成本**：不投入云端算力，毛利率 = 100%
- **无 API 流量费**：除 LLM/Vision API Key 外无费用
- **无限使用**：用户不被调用次数/并发限制

## LLM/视觉 API 处理

| 组件 | 本地/云端 | 说明 |
|------|:---:|------|
| 视频帧分析 | 云端 API | Qwen3.7 / Gemini Vision / Claude Vision |
| 解说稿生成 | 云端 API | DeepSeek / Qwen / Kimi / GLM-5 等 |
| 配音合成 | **双模式** | Edge-TTS（云端免费）/ F5-TTS（本地零样本） |
| 帧提取 | **本地** | FFmpeg + OpenCV |
| 视频合成 | **本地** | FFmpeg + 可选 CUDA |
| 情感分析 | **本地** | OpenCV + librosa |

## 后果

- ✅ 用户数据完全自主，零隐私担忧
- ✅ 商业模式简单（无服务器运营成本）
- ⚠️ 用户机器性能影响体验（需 README 提示最低配置）
- ⚠️ LLM/Vision API 仍依赖云端（不可避免）

## 替代方案探索

- **全云端渲染**: 拒绝（隐私 + 成本）
- **P2P 算力网络**: 拒绝（运维复杂度）
- **Serverless GPU (如 RunPod)**: 评估中（未来可作 P3 长期目标）
