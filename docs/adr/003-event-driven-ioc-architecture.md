# ADR-003: 事件驱动 + IoC 容器架构

- **状态**: ✅ Accepted
- **日期**: 2026-06-04
- **作者**: 架构团队
- **关联版本**: v1.0.0 ~ v2.0.0

## 背景

SceneFab 作为多功能桌面应用，模块众多（UI / 流水线 / 服务 / 数据 / 平台），需要清晰的模块解耦方案。

候选方案：
1. **事件驱动 + IoC 容器**（EventBus + ServiceContainer）
2. **直接依赖调用**（模块互相 import）
3. **回调函数 + 单例**

## 决策

采用 **EventBus（事件驱动）+ ServiceContainer（IoC 容器）** 双引擎架构。

## 关键理由

### 1. EventBus 解耦模块通信
- 业务事件（`VideoAnalyzed`、`NarrationGenerated`、`VoiceSynthesized`）通过 EventBus 广播
- 模块不直接依赖其他模块，订阅事件即可响应
- 线程安全：内部用 `threading.Lock` 保护处理器列表

### 2. ServiceContainer 统一服务管理
- **生命周期**：Singleton / Transient / Factory 三种模式
- **延迟初始化**：服务按需创建，避免启动时全量加载
- **线程安全**：内部锁保护，PySide6 + Worker 线程可安全访问
- **可测试性**：测试时可注入 mock 服务

## 已注册事件

| 事件名 | 触发时机 | 典型订阅者 |
|--------|----------|-----------|
| `video_loaded` | 视频文件导入完成 | UI 状态更新 |
| `video_analyzed` | 视频分析完成 | 情感选段模块 |
| `segments_selected` | 情感选段完成 | 脚本生成模块 |
| `script_generated` | 解说稿生成完成 | TTS 模块 |
| `voice_synthesized` | 配音合成完成 | 字幕对齐模块 |
| `captions_aligned` | 字幕对齐完成 | 视频合成模块 |
| `video_exported` | 视频导出完成 | UI 通知 + 审计日志 |
| `pipeline_failed` | 流水线失败 | 错误恢复 + UI 通知 |

## 后果

- ✅ 模块边界清晰，可独立测试
- ✅ UI 与业务逻辑完全解耦
- ✅ 未来扩展第三方插件不影响核心
- ⚠️ 事件流追踪需要日志（已规划 AuditLogger v2.0）
- ⚠️ 过多事件订阅可能影响性能（实测：千级事件下 < 5ms 延迟）

## 替代方案探索

- **直接 import**: 拒绝（耦合严重）
- **回调注册**: 评估（太底层，不易调试）
- **消息队列 (RabbitMQ/Redis)**: 拒绝（桌面应用无需分布式消息）

## 评审

- 决策贯穿 v1.0 ~ v2.0 三个大版本，已稳定
- 在 v2.0 中补充 AuditLogger 以解决事件流追踪问题
