## 🔧 优化技术详解

### 1. HTTP 连接池复用

```python
# 优化前：每次请求新建连接
requests.post(url, ...)

# 优化后：连接池复用
session = requests.Session()
adapter = HTTPAdapter(pool_connections=10, pool_maxsize=10)
session.mount('https://', adapter)
session.post(url, ...)  # 复用连接
```

### 2. 限流器信号量优化

```python
# 优化前：空转轮询
while True:
    with lock:
        if tokens >= 1.0:
            tokens -= 1.0
            return True
    time.sleep(0.01)  # CPU 100% 空转

# 优化后：信号量阻塞
semaphore = threading.Semaphore(0)
# 后台线程补充令牌
# 主线程等待信号量，不消耗 CPU
semaphore.acquire(timeout=30)
```

### 3. 批量帧处理

```python
# 优化前：逐帧处理
for ts in timestamps:
    frame = extract_frame(video, ts)  # 打开视频 1000 次
    result = analyze(frame)          # API 调用 1000 次

# 优化后：批量 + 并行
frames = extract_frames_batch(video, timestamps)  # 一次打开
with ThreadPoolExecutor(max_workers=4) as ex:
    results = list(ex.map(analyze, frames))  # 4 并行
```

### 4. LRU 缓存

```python
class LRUCache:
    def __init__(self, max_size=100):
        self.cache = OrderedDict()
    
    def get(self, key):
        if key in self.cache:
            self.cache.move_to_end(key)  # 访问时移到末尾
            return self.cache[key]
        return None
```

## 🎯 优化策略总结

### 高优先级优化
1. ✅ HTTP 连接池 - 降低延迟
2. ✅ 批量 API 调用 - 减少网络往返
3. ✅ 并行帧分析 - 利用多核
4. ✅ 限流器优化 - 减少 CPU 占用

### 中优先级优化
5. 🔄 LRU 缓存 - 减少重复计算
6. 🔄 持久化缓存 - 加速重复任务
7. 🔄 FFmpeg 复用 - 减少进程创建

8. 📋 场景检测采样 - 加速预处理
